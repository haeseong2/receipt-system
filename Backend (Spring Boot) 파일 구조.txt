Spring Boot Controller / Service / Client 구현 보조용

언제 쓰나
OcrPipelineService 흐름 짤 때
외부 API(OCR/LLM) 호출 코드 필요할 때
저장 로직 구현 전 단계

✅ Service 요청 템플릿
너는 Spring Boot 백엔드 개발자다.
[클래스 역할]
- 이 서비스는 OCR → LLM 파이프라인의 흐름만 제어한다.
- 직접 OCR 처리하거나 DB에 접근하지 않는다.

[입력]
- MultipartFile image

[출력]
- LLM이 생성한 Receipt Draft JSON

[제약 조건]
- Entity 접근 금지
- Repository 사용 금지
- 한 responsibility만 유지

[요청]
- 이 서비스의 메서드 흐름을 코드 또는 의사코드로 작성해줘

✅ Client 요청 템플릿 (외부 서버 호출)
너는 Spring에서 외부 서버를 호출하는 Client 클래스를 작성한다.

[역할]
- OCR 서버에 multipart 이미지 전송
- JSON 응답 수신

[제약]
- 비즈니스 로직 포함 금지
- URL, timeout은 설정값으로 분리

[요청]
- RestTemplate 또는 WebClient 기반 예시 코드 작성
